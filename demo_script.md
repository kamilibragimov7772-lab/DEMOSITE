# Demo Script (30 минут)

Скрипт показа клиенту. Из `TERMY_05B_Demo_Flow_and_Boundary_Rules.md` раздел 4. Готовые narration-фразы — раздел 7.

## 0:00–0:03 — Открытие (intro)

> «Сегодня покажу demo-прототип Phase 1. Это не MVP в продакшн — это рабочий скелет шести модулей, которые можно запустить на уже собранных данных. Модули Склад и Сменный выпуск — вне демо, их откроем в Phase 3 после визита на производство. Если будут вопросы — задавайте на ходу или в конце.»

## 0:03–0:08 — Flow 1: Dashboard → Catalog → SKU card

1. Открыть `/` — 4 плитки.
2. Нажать «Быстрый поиск SKU», в Topbar ввести «ТПС».
3. Клик на `ТПС 05/1` → `/catalog/PV-007`.
4. По блокам: идентификация, техспек, упаковка, применимые документы, параметры печати, claims.

**Ценность:** «Всё, что было в 5 прайсах и 3 docx — в одном месте.»

## 0:08–0:13 — Flow 2: Documents / Expiring

1. На карточке ТПС_05 кликнуть на бейдж декларации 1158277.
2. Открывается `/documents/DOC-DEC-1158277` с встроенным PDF.
3. Вернуться на `/` → плитка «Ближайшие просрочки» → клик.
4. `/documents/expiring` — видны декларации с датами 2032, 2034.

**Ценность:** «Сроки под контролем. Напоминание за 90/30/7 дней.»

## 0:13–0:18 — Flow 3: Print Check Mockup

1. На SKU-карточке в блоке «Параметры печати» → клик «→ Проверить макет».
2. `/print/check-mockup` — 12 правил.
3. Пройти несколько чекбоксов, сформировать отчёт.

**Ценность:** «Раньше проверка макета — 30 минут дизайнера. Теперь — 30 секунд.»

## 0:18–0:23 — Flow 4: Sample Request Creation

1. `/samples/new` → заполнить форму.
2. Выбрать клиент, SKU, заполнить обязательные поля.
3. Канал: СДЭК.
4. Сохранить → alert «заявка создана, записано в audit log».

**Ценность:** «Единая форма. РОП видит ожидающие одобрения.»

## 0:23–0:26 — Flow 5: Locked Modules

1. В sidebar клик на 🔒 «Склад» → `/warehouse` → прочитать message-screen.
2. Клик на 🔒 «Смены» → прочитать.

**Ценность:** «Склад и смены — Phase 3. Нужен визит на производство.»

## 0:26–0:30 — Flow 6: Audit Transparency

1. `/admin/audit` → журнал действий.
2. Фильтр по пользователю.
3. Показать, что нет кнопки «удалить».

**Ценность:** «Append-only. Ничего не теряется и не скрывается.»
