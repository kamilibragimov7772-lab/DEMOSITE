# TERMY — полировка сайта

Рабочая папка для проекта «сделать из сырого сайта TERMY конфетку».

## Структура папок

| Папка | Что сюда кладётся |
|---|---|
| `01_source_materials/` | **Сюда Камиль загружает все исходные материалы о компании:** презентации, PDF с характеристиками, брендбук, тексты, фото, прайсы, договоры, маркетинговые материалы |
| `02_github_repo/` | Склонированный сырой репозиторий сайта с GitHub |
| `03_content_draft/` | Рабочие драфты текстов, разметки, структуры страниц |
| `04_final/` | Финальный «конфетный» вариант для заливки обратно в репозиторий |
| `05_references/` | Референсы: сайты-образцы, гайдлайны, скриншоты для вдохновения |

## План работы

1. Камиль кладёт в `01_source_materials/` всё, что есть о компании
2. Предоставляет URL GitHub-репозитория сайта → клонирую в `02_github_repo/`
3. Аудит: что есть в сайте сейчас, чего не хватает
4. План «полировки» (контент, структура, тексты, SEO, UX)
5. Правки в `03_content_draft/`, переносим в `04_final/`
6. Пуш обратно на GitHub

## Что нужно от Камиля сейчас

1. **URL GitHub-репозитория** сайта (public или private — если private, понадобится gh-авторизация)
2. **Загрузить материалы** в `01_source_materials/` — можно сразу всё, что есть под рукой
3. **Сказать, что именно раздражает в текущем сайте** — это быстрее чем мне гадать (в 2 предложениях: «плохо выглядит X», «не хватает Y», «надо добавить Z»)
